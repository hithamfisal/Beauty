# Beauty Home Service v4.2 — Arabic-Only UI + Future Language Switch Ready

## الهدف
تم توحيد جميع الواجهات الحالية لتكون باللغة العربية فقط في هذه المرحلة، مع تجهيز أساس مبدئي يسمح بإضافة مفتاح تبديل اللغة عربي / إنجليزي لاحقاً بدون تغيير منطق العمل.

## ما تم تعديله
- تحويل أسماء التطبيق الظاهرة إلى العربية في تطبيق العميلة وتطبيق المدير ولوحة الويب.
- تحويل عناصر التنقل والبطاقات والفوتر في واجهة العميلة إلى العربية فقط.
- تحويل رسائل الأخطاء العامة والرسائل النصية والقوالب المرتبطة بالعميل إلى العربية.
- تحويل واجهة تسجيل دخول لوحة الإدارة والعنوان الجانبي والعلوي إلى العربية.
- تحويل اسم تطبيق Android الظاهر على الجهاز إلى العربية:
  - تطبيق العميلة: بيوتي هوم سيرفس
  - تطبيق المدير: إدارة بيوتي

## ما تم تجهيزه لاحقاً للغات
تمت إضافة ملفات أساس للترجمة المستقبلية:

```text
customer-web/src/locales/ar.js
customer-web/src/locales/en.js
admin-dashboard/src/locales/ar.js
admin-dashboard/src/locales/en.js
mobile-app/lib/i18n/app_strings.dart
admin-mobile-app/lib/i18n/admin_strings.dart
```

هذه الملفات لا تعرض مفتاح تبديل حالياً. الهدف منها تجهيز المشروع للمرحلة القادمة عندما يتم اعتماد الترجمة الإنجليزية النهائية.

## المطلوب لاحقاً لإضافة مفتاح اللغة
1. نقل كل النصوص الثابتة من الصفحات إلى ملفات الترجمة.
2. إضافة حالة لغة عامة `ar/en` في الويب و Flutter.
3. حفظ اللغة المختارة في `localStorage` للويب و `SharedPreferences` للجوال.
4. تغيير الاتجاه تلقائياً:
   - العربية: RTL
   - الإنجليزية: LTR
5. مراجعة كل النصوص الإنجليزية قبل التفعيل التجاري.

## ملاحظة
لم يتم تغيير منطق الحجز، الإدارة، النسخ الاحتياطي، أو الاتصال بالـ API. التعديل خاص باللغة والجاهزية المستقبلية للترجمة.
