# Beauty Home Service — v3.7 + v3.8

## v3.7 Admin Dashboard Redesign

تم تطوير لوحة الإدارة بصرياً ووظيفياً مع المحافظة على المنطق الحالي والـ APIs الموجودة.

### ما تم تحسينه

- Sidebar عربي RTL بتصميم أنعم ومتناسق مع هوية Luxury Soft Beauty.
- صفحة Overview جديدة تعرض:
  - كل الطلبات.
  - الطلبات الجديدة.
  - طلبات اليوم.
  - الطلبات بدون خبيرة.
  - الطلبات غير المدفوعة.
  - الخبيرات الفعالات.
- إضافة مخطط حركة الحجوزات الشهرية.
- إضافة تحليل سريع للحجوزات حسب الحالة.
- إضافة أكثر المدن طلباً.
- إضافة قائمة طلبات تحتاج متابعة مباشرة من Overview.
- تحسين Bookings Management:
  - عنوان ووصف واضحان.
  - زر إنشاء طلب مباشر.
  - فلاتر منظمة.
  - جدول احترافي مع sticky header.
  - شارات حالة Status Badges.
  - شارات دفع Payment Badges.
  - مؤشر مراحل الطلب Status Stepper داخل الجدول.
- تحسين Booking Details Modal:
  - تقسيم التفاصيل إلى أقسام: العميلة، الخدمة والموقع، الموعد والخبيرة، الدفع والملاحظات.
  - عرض حالة الطلب والدفع بشكل أوضح.
  - تغيير الحالة.
  - تعيين خبيرة.
  - تعديل الدفع.
  - سجل أحداث الطلب.
- تحسين Services / Categories / Cities / Artists بصرياً من خلال نفس نظام التصميم العام.
- تحسين Backup Page بصرياً، مع بقاء وظائف النسخ الاحتياطي كما هي.

## v3.8 UX Enhancements

### Booking Summary

تمت إضافة بطاقة ملخص داخل صفحة إنشاء الطلب قبل الحفظ، وتشمل:

- اسم العميلة.
- الجوال.
- القسم.
- الخدمة.
- الموقع.
- الموعد.
- الخبيرة المفضلة.
- السعر المتوقع من بيانات الخدمة.

### Better Status Tracking

تمت إضافة:

- Status Badge لكل حالة.
- Payment Badge لكل حالة دفع.
- Status Stepper يوضح رحلة الطلب:
  - جديد.
  - مراجعة.
  - مؤكد.
  - تعيين.
  - تنفيذ.
  - مكتمل.

### Admin Notes

تمت إضافة صندوق ملاحظة إدارية داخل تفاصيل الطلب. عند الحفظ يتم تسجيلها في booking_events عبر endpoint الموجود:

```text
POST /api/admin/bookings/:id/events
```

### Artist Assignment

تم تحسين تجربة تعيين الخبيرة داخل:

- جدول الطلبات.
- نافذة تفاصيل الطلب.

مع تحديث الحالة تلقائياً إلى `beautician_assigned` عند التعيين حسب المنطق الموجود في الـ backend.
